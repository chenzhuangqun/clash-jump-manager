# Clash Jump Manager

Clash Jump Manager is a Windows local Web UI for managing Clash Verge Rev / Mihomo `dialer-proxy` override scripts.

It helps you use one reachable subscription node as a jump proxy for another subscription whose server is not directly reachable on the current network.

The app is packaged as a single Windows executable. It starts a local HTTP server at `127.0.0.1:8766`, embeds the Web UI with Go `embed.FS`, and does not require global Python or Node.js.

## Features

- Read Clash Verge Rev `profiles.yaml`, subscription cache YAML files, and profile scripts.
- Select a jump source node from existing subscription nodes.
- Mark target subscriptions and generate `dialer-proxy` override scripts.
- Preview redacted scripts before writing.
- Back up existing scripts before overwrite.
- Detect current actual script status.
- Detect scripts generated by this tool versus third-party/manual jump scripts.
- Reset only scripts generated or managed by this tool.
- Keep local state free of node passwords, UUIDs, and other sensitive fields.

## Safety Model

This tool edits Clash Verge Rev profile script files only after user confirmation.

- Preview before write.
- Backup before overwrite.
- Reset only managed scripts.
- Do not store node passwords, UUIDs, `ws-opts`, `reality-opts`, or `plugin-opts` in `state.json`.
- Load real node secrets from the local Clash subscription cache only when applying.
- Listen only on `127.0.0.1`.
- Do not replace Clash Verge Rev, Mihomo, or the system proxy.

Clash Verge Rev config is usually located at:

```text
%APPDATA%\io.github.clash-verge-rev.clash-verge-rev
```

Important files:

```text
profiles.yaml
profiles/<uid>.yaml
profiles/<script_uid>.js
```

## Download And Run

1. Download the Windows release zip from GitHub Releases.
2. Extract it to a writable folder, such as Desktop, Downloads, or a tools folder.
3. Double-click `clash-jump-manager.exe`.
4. Open `http://127.0.0.1:8766/` if the browser does not open automatically.

Do not place the executable under `C:\Program Files` unless you know the folder is writable. The portable build writes `state.json` and `backups/` beside the executable.

## Build From Source

Requirements:

- Windows
- Go 1.26 or newer

Run locally:

```powershell
go run ./cmd/clash-jump-manager
```

Build:

```powershell
go build -o dist/clash-jump-manager.exe ./cmd/clash-jump-manager
```

Test:

```powershell
go test ./...
```

If the default Go module proxy is unavailable:

```powershell
$env:GOPROXY='https://goproxy.cn,direct'
go test ./...
```

## Project Structure

```text
cmd/clash-jump-manager/      # executable entrypoint and embedded Web UI
internal/clash/              # Clash Verge profiles, subscriptions, node cache
internal/jump/               # match rules, script generation, script inspection
internal/store/              # local state, backups, managed reset boundaries
internal/server/             # HTTP API and static Web server
static/                      # plain HTML/CSS/JS reference copy
```

## API

Main read APIs:

```text
GET    /api/subscriptions
GET    /api/subscriptions/{uid}/nodes
GET    /api/jump/state
GET    /api/jump/preview
POST   /api/jump/preview
GET    /api/runtime/status
GET    /api/diagnostics
```

Write APIs:

```text
POST   /api/apply
POST   /api/reset
POST   /api/scripts/{uid}/disable-foreign-jump
```

Legacy state mutation APIs still exist for compatibility, but the Web UI keeps edits as a local draft until preview/apply.

## Release Packaging

For a portable release, package only:

```text
clash-jump-manager.exe
README.txt
```

Do not publish:

```text
state.json
backups/
dist/
*.zip generated on your machine
Clash Verge subscription cache files
```

Backups and subscription cache files may contain real node secrets.
